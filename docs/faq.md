# 常见问题

## 这只适合小说吗？

不是。它最初很适合长篇写作，但框架本身是通用的。你可以用它管理研究、文章、产品文档、知识库、课程或长期创作项目。

## 必须使用 Obsidian 吗？

不必须，但它按 Obsidian 友好的 Markdown 文件和 Vault 目录设计。

## 必须使用 Claudian 吗？

不必须。Claude、Claude Code、Claudian 或任何能读取 Markdown 项目文件的 Agent 都可以使用。

## 为什么要把写作、审稿、更新拆成不同 Skill？

因为不同阶段的目标不同。写作需要流动性，审稿需要挑错，更新项目记忆需要保守。拆开之后，长项目更稳定。

## 可以改目录名吗？

可以。请在 `00_System/ProjectConfig.md` 里写清楚目录映射。

## 所有模板都要复制吗？

新项目建议全部复制。已有 Vault 可以只复制系统文件和需要的 Skill，然后逐步迁移。

## 可以公开基于这个框架的 Vault 吗？

可以，但请先移除私人笔记、API Key、未公开稿件、私有项目数据和未授权源材料。
