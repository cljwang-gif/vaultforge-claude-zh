# 架构说明

`vaultforge-claude-zh` 把一个 Obsidian Vault 拆成几个职责清晰的层。

## 分层结构

```text
CLAUDE.md
  根级总控协议。定义权限、任务路由、上下文读取、审查、更新和停止规则。

00_System/
  单一事实源。保存项目配置、当前状态、写作规则、语言规则、平台规则和索引表。

09_SKILLS/
  岗位行为。每个 Skill 只负责一个阶段，例如规划、写作、审稿、项目更新或审计。

03_Plot/
  项目逻辑。追踪主线、支线、伏笔、新增信息和矛盾。

05_Outputs/
  产出物。保存章节、文章、报告、规格、草稿等。

06_Review/
  质量记录。保存审稿、评分、更新记录和审计报告。
```

## 为什么这样设计

长项目最容易坏在三件事：

- 上下文读取过多，模型混淆事实。
- 写作、审稿、更新记忆混在一起。
- 临时信息被过早升级成正式设定。

VaultForge 通过职责拆分降低这些风险：

- 规则和产出分离。
- 当前事实和未来计划分离。
- 写作和审稿分离。
- 审稿和项目更新分离。
- 临时信息和正式设定分离。

## 推荐闭环

```text
读取总控协议
→ 判断任务类型
→ 加载最小上下文
→ 生成或检查当前对象
→ 通过质量门槛
→ 更新项目记忆
→ 停止等待授权
```

## 目录改名

你可以改目录名，但要保留职责。

研究项目可以这样映射：

```text
03_Plot/       -> 03_ResearchLogic/
05_Outputs/    -> 05_Drafts/
02_Characters/ -> 02_People/
```

产品项目可以这样映射：

```text
03_Plot/     -> 03_ProductLogic/
04_Plans/    -> 04_Roadmap/
05_Outputs/  -> 05_Specs/
```
