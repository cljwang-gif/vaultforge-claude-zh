# WriteChapter

## 职责

根据已批准规划或明确用户请求，生成当前授权产出。

本 Skill 可用于章节、文章、报告、规格、论文草稿、课程文本等交付物。

## 权限

应该：

- 读取当前规划和直接相关上下文。
- 生成当前授权产出。
- 遵守项目写作规则和语言规则。
- 捕获新增信息，留给后续项目更新。

不得：

- 更新 Registry 或项目长期记忆。
- 重写无关文件。
- 为补上下文而发明重大事实。
- 开始下一个产出。
- 未经要求把审稿意见混入最终产出。

## 上下文

读取：

- `CLAUDE.md`
- `00_System/ProjectConfig.md`
- `00_System/ProjectState.md`
- `00_System/WritingRules.md`
- `00_System/LanguageRules.md`
- 必要时读取 `00_System/PlatformRules.md`
- 当前规划或任务说明
- 只读取直接相关的人物、实体、线程、来源和历史产出

## 硬指标

- 符合请求范围。
- 符合格式和长度要求。
- 不输出无依据断言。
- 必要假设必须标注。
- 不在产出中更新长期记忆。

## 输出格式

优先使用项目已有产出模板。

若无模板，使用：

```markdown
---
类型: 产出
状态: 草稿
更新日期: YYYY-MM-DD
---

# <标题>

<正文>

<!-- 待 UpdateProject 捕获新增信息
- 人物：
- 实体：
- 地点：
- 规则：
- 开放问题：
-->
```

## 停止条件

生成当前授权产出并报告路径或内容后停止。不得自动进入下一产出。
